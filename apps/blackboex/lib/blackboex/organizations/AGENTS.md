# Organizations ‚Äî Agent Context

Multi-tenancy layer. Every resource (APIs, API keys, memberships, billing) belongs to an organization. Users access resources only through their organization memberships, mediated by the `Membership` join schema. The context facade is `Blackboex.Organizations` (`../organizations.ex`).

---

## Organization Schema

File: `organization.ex` ‚Äî Module: `Blackboex.Organizations.Organization` ‚Äî Table: `organizations`

| Column       | Type        | Constraints              | Notes                         |
|--------------|-------------|--------------------------|-------------------------------|
| `id`         | `binary_id` | PK, autogenerated        | UUID                          |
| `name`       | `string`    | NOT NULL                 | Human-readable display name   |
| `slug`       | `string`    | NOT NULL, UNIQUE         | URL-safe identifier           |
| `plan`       | `string`    | NOT NULL, default "free" | `:free`, `:pro`, `:enterprise` |
| `inserted_at`| `utc_datetime` |                       |                               |
| `updated_at` | `utc_datetime` |                       |                               |

Associations: `has_many :memberships`, `has_many :users, through: [:memberships, :user]`

**Slug generation:** derived from `name` when not provided ‚Äî lowercased, non-alphanumeric stripped, spaces collapsed to hyphens. Validated: required, 1‚Äì100 chars, format `^[a-z0-9]([a-z0-9-]*[a-z0-9])?$`, unique index.

**Changesets:** `changeset/2` (standard), `admin_changeset/3` (Backpex, third arg `_metadata` ignored). Slug is editable but treat as immutable in practice ‚Äî changing it silently breaks `/p/:org_slug/:api_slug` URLs.

---

## Membership Schema

File: `membership.ex` ‚Äî Module: `Blackboex.Organizations.Membership` ‚Äî Table: `memberships`

| Column            | Type        | Constraints                  | Notes                     |
|-------------------|-------------|------------------------------|---------------------------|
| `id`              | `binary_id` | PK, autogenerated            |                           |
| `role`            | `string`    | NOT NULL                     | `:owner`, `:admin`, `:member` |
| `user_id`         | `integer`   | FK ‚Üí users(id), NOT NULL     | Integer FK (users PK is integer) |
| `organization_id` | `binary_id` | FK ‚Üí organizations(id), NOT NULL |                       |
| `inserted_at`     | `utc_datetime` |                           |                           |
| `updated_at`      | `utc_datetime` |                           |                           |

Indexes: `[user_id]`, `[organization_id]`, `unique [user_id, organization_id]`.

Cascade: deleting a user or organization cascade-deletes memberships. Deleting an organization does NOT cascade to APIs or other resources unless those migrations declare `on_delete: :delete_all`.

### Roles

| Role      | Description                                                                               |
|-----------|-------------------------------------------------------------------------------------------|
| `:owner`  | Full control ‚Äî manage members, delete org, manage billing                                |
| `:admin`  | Create/update/delete APIs, manage members and API keys                                    |
| `:member` | Read access + create/update APIs and run tests/docs. Cannot delete APIs or manage members |

**Mixed FK types:** `user_id` is `type: :id` (integer); `organization_id` is `:binary_id`. This asymmetry must be preserved in any new schema referencing both.

## Public API

All functions in `Blackboex.Organizations`.

```elixir
create_organization(User.t(), map()) ::
  {:ok, %{organization: Organization.t(), membership: Membership.t()}}
  | {:error, atom(), Ecto.Changeset.t(), map()}
```
Atomic `Ecto.Multi` ‚Äî creates org + owner membership. No way to create an org without creating the first membership.

```elixir
list_user_organizations(User.t()) :: [Organization.t()]
get_organization!(Ecto.UUID.t()) :: Organization.t()   # raises on not found
get_organization(Ecto.UUID.t()) :: Organization.t() | nil
```

```elixir
add_member(Organization.t(), User.t(), atom()) :: {:ok, Membership.t()} | {:error, Ecto.Changeset.t()}
```
Fires async audit log `"member.added"` via `Blackboex.LoggingSupervisor`. Returns `{:error, changeset}` including unique constraint violation if user is already a member.

```elixir
get_user_membership(Organization.t(), User.t()) :: Membership.t() | nil
get_user_primary_plan(User.t()) :: atom()
```
`get_user_primary_plan/1` returns the plan of the user's oldest org. Falls back to `:free`. Used only by `Features.GroupImpl` ‚Äî not for authorization checks.

## Multi-tenancy Rules

1. **No cross-org access.** Every resource fetch must include `organization_id`. Bare ID lookup without org check is an IDOR vulnerability.
2. **Membership is the gate.** The `Accounts.Scope` carries `user` + `organization`. The `SetOrganization` on_mount hook sets the scope from session.
3. **Slug is the public URL key.** `/p/:org_slug/:api_slug` ‚Äî treat as stable once set.
4. **Cascade on delete.** Memberships are cascade-deleted when user or org is deleted. No soft-delete.
5. **Personal org is not special in the schema.** No `is_personal` flag. Tests must not assume a user has exactly one org.

---

## Authorization

Handled by `Blackboex.Policy` (LetMe DSL). See `apps/blackboex/lib/blackboex/policy/AGENTS.md` and `policy.ex`. The scope's `role` drives all policy checks. Call `Policy.authorize/3` or `Policy.authorize_and_track/3` (the latter emits `[:blackboex, :policy, :denied]` telemetry).

---

## Testing

`AccountsFixtures.user_fixture/0` is the entry point ‚Äî it calls `register_user/1` which creates user + personal org + owner membership atomically. No ExMachina factories for this context. Use `DataCase` with `async: true`, tag `@moduletag :unit`.

To build a second org or add a member, call `Organizations.create_organization/2` and `Organizations.add_member/3` directly in test setup.

---

## Gotchas

**Slug collisions at registration.** Personal org name = `email_prefix + random_6char_suffix`. The registration `Multi` fails at `:organization` on collision ‚Üí collapses to `{:error, :registration_failed}`.

**`create_organization/2` rolls back entirely on membership failure.** The Multi inserts org first, then membership. Any membership failure rolls back the org insert.

**`add_member/3` returns `{:error, changeset}` for duplicate membership.** Unique index on `(user_id, organization_id)`.

**Integer vs UUID FK on `user_id`.** `memberships.user_id` is an integer. Supply an integer `user_id`, not a UUID string, or Ecto will fail with a type error.

**`get_user_primary_plan/1` is not a membership check.** Its only consumer is `Features.GroupImpl`. For authorization, use `get_user_membership/2`.

**`list_user_organizations/1` does not preload memberships.** Call `get_user_membership/2` separately if role information is needed.
